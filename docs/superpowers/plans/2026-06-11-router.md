# Real Routing for Cairn — Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Make the vestigial `react-router-dom` v7 dependency earn its place by giving Cairn real routing — the active note/graph/tag lives in the URL, enabling deep links and Obsidian-style back/forward history — without disturbing the Zustand store's ownership of tabs and content.

**Architecture:** The URL owns one fact (what is foregrounded: a note, the graph, or a tag filter). The store keeps owning tabs, buffers, autosave, search, persistence. A single `RouteSync` controller reconciles the two with two guarded lanes (URL→store, store→URL) whose decision logic lives in pure, unit-tested functions. `HashRouter` for Tauri reload-safety. All user navigation flows through `navigate()`.

**Tech Stack:** React 18, react-router-dom 7, Zustand, Vitest + Testing Library, TypeScript.

**Spec:** `docs/superpowers/specs/2026-06-11-router-design.md`

**Working branch:** `feat-router` (already created off latest `main`).

```ts
/** Minimal shape we need from react-router's Location (and easy to test). */
export interface RouteLocation {
  pathname: string;
}

/** Build the URL for a note path (each segment encoded, slashes preserved). */
export function noteUrl(path: string): string {
  return "/note/" + path.split("/").map(encodeURIComponent).join("/");
}

/** Extract a note path from a `/note/*` location, or null if not one. */
export function notePathFromLocation(loc: RouteLocation): string | null {
  const m = loc.pathname.match(/^\/note\/(.+)$/);
  if (!m) return null;
  return m[1].split("/").map(decodeURIComponent).join("/");
}

/** Build the URL for a tag filter (tag fully encoded — slashes too). */
export function tagUrl(tag: string): string {
  return "/tags/" + encodeURIComponent(tag);
}

/** Extract a tag from a `/tags/:tag` location, or null if not one. */
export function tagFromLocation(loc: RouteLocation): string | null {
  const m = loc.pathname.match(/^\/tags\/([^/]+)$/);
  return m ? decodeURIComponent(m[1]) : null;
}

/** True only on the graph route. */
export function isGraph(loc: RouteLocation): boolean {
  return loc.pathname === "/graph";
}
```

## Task 2: Pure reconciliation logic (`routeSync.ts`)

This is the heart of the design — the two lanes as pure functions. `urlToStore` returns an ordered list of effects to apply to the store given the current URL. `storeToUrl` returns at most one navigation given an `activePath` change. The `prevActivePath` argument is what lets `storeToUrl` distinguish a store-origin change (URL trailing — reconcile) from a user/deep-link navigation (URL leading — stay out so the URL→store lane handles it).

```ts
import {
  type RouteLocation,
  noteUrl,
  notePathFromLocation,
  tagFromLocation,
  isGraph,
} from "./routes";

export interface UrlToStoreInputs {
  location: RouteLocation;
  activePath: string | null;
  activeTag: string | null;
  searchActive: boolean;
}

export type UrlToStoreEffect =
  | { kind: "filterByTag"; tag: string }
  | { kind: "closeSearch" }
  | { kind: "loadGraph" }
  | { kind: "openNote"; path: string };

/** Lane A: what the store should do to match the current URL. */
export function urlToStore(i: UrlToStoreInputs): UrlToStoreEffect[] {
  const tag = tagFromLocation(i.location);
  if (tag !== null) {
    return i.activeTag === tag ? [] : [{ kind: "filterByTag", tag }];
  }

  const effects: UrlToStoreEffect[] = [];
  // Leaving a tag/search overlay: clear it before showing a note or the graph.
  if (i.searchActive || i.activeTag !== null) {
    effects.push({ kind: "closeSearch" });
  }
  if (isGraph(i.location)) {
    effects.push({ kind: "loadGraph" });
    return effects;
  }
  const notePath = notePathFromLocation(i.location);
  if (notePath !== null && notePath !== i.activePath) {
    effects.push({ kind: "openNote", path: notePath });
  }
  return effects;
}

export interface StoreToUrlInputs {
  location: RouteLocation;
  activePath: string | null;
  prevActivePath: string | null;
}

export type StoreToUrlEffect =
  | { kind: "navigate"; to: string }
  | { kind: "none" };

/** Lane B: how the URL should follow a store-origin activePath change. */
export function storeToUrl(i: StoreToUrlInputs): StoreToUrlEffect {
  // Graph and tag routes are explicit, user-chosen foregrounds — never override.
  if (isGraph(i.location) || tagFromLocation(i.location) !== null) {
    return { kind: "none" };
  }

  const urlNote = notePathFromLocation(i.location);
  if (i.activePath === urlNote) return { kind: "none" };

  // A note is active but the URL is at root (or non-note) → reflect it.
  if (urlNote === null) {
    return { kind: "navigate", to: noteUrl(i.activePath as string) };
  }

  // The URL names a note that isn't active. If it matches the PREVIOUS active
  // note, the URL is trailing a store-origin change (closeTab/rename/delete) →
  // reconcile. Otherwise the URL is leading (deep link / user nav) and Lane A
  // will open it — stay out to avoid fighting it.
  if (urlNote === i.prevActivePath) {
    return {
      kind: "navigate",
      to: i.activePath !== null ? noteUrl(i.activePath) : "/",
    };
  }
  return { kind: "none" };
}
```

```tsx
import { useEffect, useRef } from "react";
import { useLocation, useNavigate } from "react-router-dom";
import { cairnStore, useCairn } from "./cairnStore";
import { urlToStore, storeToUrl } from "./routeSync";

/**
 * The single place the URL and the store meet. Renders nothing.
 * Lane A reacts to URL changes and drives the store; Lane B reacts to
 * store-origin activePath changes and updates the URL. Both are guarded so they
 * converge in one step (see routeSync.ts).
 */
export function RouteSync(): null {
  const location = useLocation();
  const navigate = useNavigate();
  const activePath = useCairn((s) => s.activePath);
  const cairnPath = useCairn((s) => s.cairnPath);

  // Current location read by Lane B without making it a dependency.
  const locationRef = useRef(location);
  locationRef.current = location;
  // Previous activePath, so Lane B can tell store-origin changes from URL-led ones.
  const prevActiveRef = useRef<string | null>(activePath);

  // Lane A: URL -> store
  useEffect(() => {
    if (cairnPath === null) return;
    const st = cairnStore.getState();
    const effects = urlToStore({
      location,
      activePath: st.activePath,
      activeTag: st.activeTag,
      searchActive: st.searchResults !== null,
    });
    for (const e of effects) {
      if (e.kind === "filterByTag") void st.filterByTag(e.tag);
      else if (e.kind === "closeSearch") st.closeSearch();
      else if (e.kind === "loadGraph") void st.loadGraph();
      else if (e.kind === "openNote") void st.openNote(e.path);
    }
  }, [location, cairnPath]);

  // Lane B: store -> URL
  useEffect(() => {
    const prev = prevActiveRef.current;
    prevActiveRef.current = activePath;
    if (cairnPath === null) return;
    const eff = storeToUrl({
      location: locationRef.current,
      activePath,
      prevActivePath: prev,
    });
    if (eff.kind === "navigate") navigate(eff.to, { replace: true });
  }, [activePath, cairnPath, navigate]);

  return null;
}
```

## Notes for the implementer

- **Loop-safety:** the two lanes never echo because each is guarded by exact inequality and the `prevActivePath` trick keeps Lane B out of URL-led navigations. If you ever see a navigation loop, the bug is almost certainly a missing/incorrect `prevActivePath` update in `RouteSync` Lane B — verify `prevActiveRef.current` is reassigned every time the effect runs, before the early `cairnPath === null` return path is considered (it is, in the given code: assignment precedes the guard).
- **Why `replace` in Lane B:** store-origin changes (closeTab, restore, rename) should not create forward-history entries the user never navigated to. User navigations use the default `push` (via `navigate(noteUrl(p))` with no options).
- **Don't** reintroduce `loadGraph()` calls in `App.tsx` — RouteSync owns graph loading on `/graph` entry now.
- **HashRouter URLs** look like `/#/note/...`; `useLocation().pathname` is still `/note/...`, so all `routes.ts` helpers work unchanged.
- **Unknown note in a deep link (known simplification):** `MockClient.get_note` throws `not_found`, which `openNote` catches and surfaces via the existing error toast — so a bad `/note/x` shows the error and leaves the prior view in place. We deliberately do **not** also redirect to `/` (the spec mentioned it): that would require `openNote` to report failure back to `RouteSync`, adding cross-layer plumbing for a rare case. Error-toast-and-stay is the simpler, acceptable behavior. Flag to the user if they want the redirect.
